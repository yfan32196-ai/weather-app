## 总体设计

①气象记录管理
- 添加记录：增加新的气象数据，录入城市、日期、温度、湿度、气压、风速、风向、天气状况等信息，自动写入数据文件。
- 查询记录：通过城市、时间范围、数据来源等多种方式筛选记录，支持按时间、温度排序。
- 修改记录：通过记录ID查找目标记录，若找到则允许修改温度、湿度、风速等字段，否则提示无该记录。
- 删除记录：通过记录ID查找目标记录，若找到则删除，否则提示无该记录。

②城市管理
- 添加城市：增加新的城市信息，录入名称、省份、经纬度，检查城市名是否已存在，若存在则取消添加并提示。
- 编辑城市：通过城市ID查找目标城市，修改名称、省份、经纬度等信息。
- 删除城市：通过城市ID查找目标城市，若找到则允许删除，删除时联动删除该城市下的所有气象记录。

③数据导入导出
- CSV导入：读取CSV文件，逐行解析城市名、时间、温湿度等字段，通过城市名匹配城市ID，批量写入数据文件。
- 手动录入：逐条从键盘输入气象数据，输入城市ID为0时结束录入。
- 数据导出：将全部气象记录导出为CSV文件，包含城市、时间、温度、湿度、风速、风向、天气、来源等完整字段。

④统计分析
- 城市概览：展示所有城市的平均温度、极值温度、平均湿度、主导风向、主要天气。
- 极端预警：扫描全部数据，检测高温(>=35°C)、强风(>=10.5m/s)、暴风雨风险并提示。

⑤用户管理
- 管理员可以添加新用户，设置用户名、密码、角色（管理员/观察者）。
- 系统启动时要求输入用户名密码，根据角色显示对应权限菜单。

### 4.2.1 添加气象记录实现

**主题思想：**

将用户输入的各项气象参数组装为一条完整的WeatherRecord结构体，存入全局记录数组，并同步写入二进制文件，实现数据的即时持久化。

**流程图：**

```
输入参数(城市ID/温度/湿度/气压/风速/风向/天气等)
    │
    ▼
记录数>=5000? ——是——→ 返回0(失败)
    │否
    ▼
find_city_by_id() 查找城市
    │
城市不存在? ——是——→ 返回0(失败)
    │否
    ▼
赋值各字段: id=next_record_id++, 温度/湿度/气压/风速/...
    │
    ▼
record_count++, 数组新增一条记录
    │
    ▼
update_city_record_count() 更新城市关联计数
    │
    ▼
save_records() 写入 records.dat
    │
    ▼
返回1(成功)
```

**函数：**

```
int add_record();                 // [添加记录] 将温度/湿度/气压/风速/风向/天气等参数组装为结构体存入数组
City *find_city_by_id();          // [查找城市] 按ID线性遍历cities数组  返回城市指针  失败返回NULL

int get_int_input();              // [整数输入] 带范围校验的键盘整数输入函数

### 4.2.2 删除气象记录实现

**主题思想：**

在数组中定位目标记录，使用memmove将后续元素整体前移覆盖，实现无空洞的数组删除，同时更新关联城市的记录计数和文件。

**流程图：**

```
输入记录ID
    │
    ▼
find_record_by_id() 遍历 records 数组
    │
未找到? ——是——→ 返回0(失败)
    │否
    ▼
计算索引 idx = 记录指针 - records数组首地址
    │
    ▼
memmove(records[idx], records[idx+1],
        (record_count-idx-1)*sizeof(WeatherRecord))
    │
    ▼
record_count--
    │
    ▼
update_city_record_count() → save_records() → save_cities()
    │
    ▼
返回1(成功)
```

**函数：**

```
int delete_record();               // [删除记录] 按ID查找后memmove前移后续记录  数组count减1
WeatherRecord *find_record_by_id(); // [查找记录] 按ID线性遍历records数组  返回记录指针  失败返回NULL
void update_city_record_count();   // [更新计数] 重新统计城市关联记录数
int save_records();                // [保存记录] fwrite将records数组写入records.dat
int save_cities();                 // [保存城市] fwrite将cities数组写入cities.dat
void list_records();               // [列出记录] 分页展示气象记录  支持按时间/温度/城市排序
void filter_records();             // [筛选记录] 按城市ID/时间范围/数据来源复合条件过滤
int count_records_by_city();       // [计数] 遍历records统计指定城市的记录总数

```

**结果：**

### 4.2.3 统计分析实现

**主题思想：**

遍历全部气象记录，按城市分组累加计算温度、湿度、风速的均值与极值，统计风向和天气频次，根据阈值输出极端天气预警。

**流程图：**

```
输入城市ID
    │
    ▼
遍历 records 数组，筛选 city_id 匹配的记录
    │
    ▼
累加 temp_sum, humid_sum, wind_sum
记录 temp_min, temp_max, wind_max
统计 wind_dir_count[8], weather_count[9]
    │
    ▼
计算平均值: avg = sum / count
    │
    ▼
找出 max(wind_dir_count) → 主导风向
找出 max(weather_count) → 主要天气
    │
    ▼
判断预警: temp_max>=35°C?  wind_max>=10.5?  雷暴?
    │
    ▼
返回 CityStats 结构体
```

**函数：**

```
CityStats compute_city_stats();     // [单城市统计] 累加求和计算均温/极值/平均湿度/风速/主导风向/主要天气
CityStats compute_all_stats();      // [全部统计] 遍历所有城市  调用compute_city_stats()汇总结果

**结果：**

返回CityStats结构体，包含平均温度、最高/最低温度、平均湿度、平均/最大风速、主导风向、主要天气以及三项预警标志。由show_stats()和extreme_weather_alert()调用展示。

### 4.2.4 文件读写实现

**主题思想：**

采用fwrite/fread二进制块读写方式，启动时一次性加载全部数据到内存，退出时一次性写回，运行期间所有操作在内存中完成，兼顾效率与数据安全。

**流程图：**

```
[启动]                              [退出/变更]
   │                                     │
   ▼                                     ▼
fopen("cities.dat","rb")          fopen("cities.dat","wb")
fread(&count, sizeof(int),1)      fwrite(&count, sizeof(int),1)
fread(cities, sizeof(City),count) fwrite(cities, sizeof(City),count)
fclose()                          fclose()
   │                                     │
   ▼                                     ▼
同逻辑加载 records.dat、users.dat   同逻辑保存三个文件
```

**函数：**

```
int init_data_dir();               // [创建目录] 使用mkdir创建data/数据存储目录
int load_all_data();                // [加载全部] 依次调用内部load函数从三个.dat文件fread加载数据
int save_all_data();                // [保存全部] 依次调用save_users/save_cities/save_records写回文件
int save_users();                   // [保存用户] 先写user_count(int)  再fwrite users数组
int save_cities();                  // [保存城市] 先写city_count(int)  再fwrite cities数组
int save_records();                 // [保存记录] 先写record_count+next_id  再fwrite records数组
void log_operation();               // [操作日志] fopen("system.log","a")追加带时间戳的操作记录
int main();                         // [主入口] 启动时调用load_all_data()  退出时调用save_all_data()
int login();                        // [登录] 验证成功后调用log_operation记录登录日志
void logout();                      // [登出] 清除状态后调用log_operation记录退出日志
void init_default_users();          // [初始用户] 首次运行创建admin和viewer  调用save_users持久化
void init_default_cities();         // [初始城市] 首次运行创建8个默认城市  调用add_city写入
```

**结果：**

程序启动时从data/目录下的三个.dat文件恢复所有数据，程序退出时将AppState中的数组完整写入文件。system.log以文本追加方式记录每次登录、登出、增删操作及时间戳，用于操作追溯。

## 程序设计相关技术

C语言是一门面向过程的计算机编程语言，具有高效、灵活、表达力强和可移植性好等特点，广泛应用于系统软件和嵌入式开发。C语言提供丰富的数据类型和控制结构，能够以较小的代码量实现复杂逻辑，且编译后的程序运行效率高，是高校计算机教学中首选的入门语言之一。

结构体是C语言中一种聚合数据类型，允许将不同类型的数据组合成一个整体。本系统中使用结构体封装气象记录的18个属性字段，包括温度、湿度、气压、风速等数值型和字符型成员，通过成员运算符访问各个字段，实现了气象数据的逻辑组织与管理。

全局变量是在所有函数之外定义的变量，从定义处开始到文件结束均可访问。本系统使用全局结构体数组 `cities`、`records`、`users` 存储所有数据，各模块函数通过指针参数 `AppState *app` 共享和操作这些数据，避免了频繁的参数传递，提高了模块间的数据交换效率。

## 设计中的难点及其解决方法

（1）在实现删除城市功能时，需要同时删除该城市下的所有气象记录。最初设计时未考虑到这一联动关系，后来修改delete_city函数，增加遍历records数组并删除匹配记录的逻辑。由于记录存储在数组中，删除需要使用memmove将后续元素前移，涉及多次数组操作，改动量较大。通过从后向前遍历的方式避免了索引错乱问题。

（2）设计文件持久化功能时，需要在程序启动时加载数据、退出时保存数据、每次增删改后即时写回文件。如何平衡IO性能和代码复杂度是一个难点。最终采用启动加载全部数据到内存、操作期间仅在关键变更后写回、退出时统一保存的策略，既保证了数据安全，又避免了频繁IO。

（3）在实现用户登录和角色权限控制时，需要根据管理员和观察者两种角色动态显示不同菜单，并在每个功能入口处校验权限。设计时将角色判断逻辑集中在主菜单循环和各功能模块入口，使用统一的role变量控制菜单选项显示，避免了权限校验代码的重复分散。

（4）CSV数据导入功能中，需要注意中文字符编码兼容性和不同日期格式的解析。通过尝试多种编码（utf-8、gbk、gb2312）依次解析，并对多种日期格式（YYYY-MM-DD HH:MM:SS、YYYY/MM/DD等）逐一尝试匹配，提高了导入的成功率和容错性。

（5）统计分析函数需要计算主导风向和主要天气，涉及对8个风向和9种天气类型的频次统计。最初使用多个if-else判断，代码冗长。后改用数组计数方式，将风向和天气的枚举值直接作为数组下标累加，最后遍历数组找最大值对应的索引，代码简洁且易于扩展。

## 不足之处

（1）气象记录结构体中，缺少对观测站或观测设备的记录，仅记录了所属城市。将来可以增加观测站信息字段，使数据来源更加精确。

（2）用户密码目前以明文形式存储在二进制文件中，安全性不足。后续应使用MD5等哈希算法对密码加密存储，提高系统安全性。

（3）系统的数据容量受限于静态数组大小（城市50个、记录5000条），当数据量超出上限时无法自动扩展。后续可改用动态内存分配（malloc/realloc）或链表结构，使容量更加灵活。

（4）统计分析目前仅实现了基本的均值、极值和频次统计，缺少更丰富的数据可视化图表和长期趋势分析功能。在C语言环境下实现图形界面有一定难度，可通过导出数据到Excel后利用图表功能间接实现。

（5）在使用C语言涉及更多底层功能时，如动态内存管理、多文件复杂交互等，掌握还不够熟练，需要更进一步的学习和实践。

## 开发环境

GCC（GNU Compiler Collection）是GNU项目开发的编译器套件，支持C、C++等多种语言，是Linux/Unix系统下标准的C语言编译器。在macOS系统中，Apple Clang基于LLVM架构，提供了与GCC兼容的命令行接口。本系统使用gcc编译器，在终端下通过Makefile自动化编译，编译命令为 `gcc -o weather_system main.c auth.c city.c weather.c stats.c file_io.c -lm`。系统源代码不依赖任何第三方库，仅使用C标准库和POSIX标准接口，具有良好的可移植性，可在安装GCC的任何平台编译运行。
