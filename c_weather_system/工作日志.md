# 气象信息管理系统课程设计工作日志

## 第一周日志

| 时间 | 工作内容及收获 |
|------|---------------|
| 第一周周一 | 今天，老师安排了C语言课程设计的任务，我确定了把气象信息管理系统作为我的课设主题。下午我去图书馆查阅了有关C语言文件操作和结构体的资料，深入了解了管理系统的模块化设计思路和二进制文件持久化存储的基本方法。 |
| 第一周周二 | 第二天，我通过网络搜索，在CSDN和博客园上查找了气象数据管理的相关案例，了解了气象管理系统的基本功能模块，包括数据录入、查询统计、文件存储等核心功能，并将大体框架罗列出来。此外，我还查找了中国主要城市的经纬度数据，为初始化城市列表做准备。 |
| 第一周周三 | 第三天，我在电脑上安装了GCC编译器和VS Code编辑器，配置了C语言开发环境。在完成环境配置后，我使用结构体和枚举设计了系统的核心数据结构，包括用户、城市、气象记录三个结构体，以及天气状况、风向等枚举类型，完成了系统的数据模型设计。 |
| 第一周周四 | 第四天，在需求分析完成后，我搭建了系统的模块架构，将系统划分为主程序模块、用户认证模块、城市管理模块、气象记录模块、统计分析模块和文件读写模块六个部分。同时完成了头文件weather_system.h的编写，定义了所有结构体、共用体、枚举和函数声明。 |
| 第一周周五 | 周五，我开始编写各模块的实现代码。完成了主菜单界面的设计和用户登录验证功能，编写了城市增删改查函数和气象记录的添加函数。在文件读写模块中，使用fwrite和fread实现了二进制文件的数据持久化存储。 |
| 第一周周六 | 周六继续完善代码，完成了气象记录的查询、修改、删除功能，实现了冒泡排序和分页展示。同时编写了统计分析模块，使用循环遍历计算各城市的平均温度、极值温度、主导风向和主要天气，并实现了极端天气预警功能。 |
| 第一周周日 | 周日进行代码整合和初步测试，将六个模块通过makefile统一编译。运行时发现字符串输入存在换行符残留问题，查阅资料后使用去除行尾换行符的方式解决。同时修正了多处菜单跳转逻辑的bug。 |

## 第二周日志

| 时间 | 工作内容及收获 |
|------|---------------|
| 第二周周一 | 第二周第一天，我继续完善系统功能。添加了CSV文件导入导出功能，使用fscanf解析CSV格式文件，通过城市名匹配实现批量导入。在手动导入方面，增加了逐条键盘录入的功能，输入城市ID为0时自动结束录入。 |
| 第二周周二 | 第二天，重点优化了用户体验。改进了菜单界面的中英文显示，添加了清屏和暂停等待功能，适配了Windows和macOS双平台。同时在文件读写模块中增加了操作日志功能，记录每次登录、登出和数据变更操作。运行测试中发现了memmove删除记录时数组越界的问题，修正了内存移动的字节数计算。 |
| 第二周周三 | 第三天，完成全部代码调试和功能验证。对系统进行了完整测试，包括添加8个默认城市、录入多条气象记录、执行统计分析、测试CSV导入导出、验证文件持久化（退出后重新启动数据不丢失）。所有功能运行正常。下午，我开始撰写课程设计报告。 |
| 第二周周四 | 第四天，继续完善实验报告。完成了项目背景、需求分析、数据需求、总体设计、数据结构设计等章节的撰写。使用Visio绘制了系统功能模块图和main函数流程图。整理了30个主要函数的声明和调用关系表格。 |
| 第二周周五 | 第五天，完成实验报告的剩余章节，包括四个核心功能的详细实现（添加记录、删除记录、统计分析、文件读写），每个功能包含主题思想、流程图、函数声明和结果描述。对报告进行全面完善，检查格式和内容的一致性，准备答辩材料。 |

**总结：** 通过本次课程设计，我掌握了结构体、枚举、共用体等C语言构造类型的实际应用，深入理解了文件操作（fread/fwrite）、模块化编程、指针传参、数组管理等核心知识。系统实现了气象数据的增删改查、统计分析、导入导出、用户权限控制等完整功能，总计约1500行代码，6个源文件模块，30个函数。通过独立完成从需求分析到编码测试的全过程，锻炼了软件工程思维和问题解决能力。
